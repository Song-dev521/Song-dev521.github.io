# 实战经验 · 大一 · 后端开发

这个暑假，我接手了一个校级大创项目的后端开发任务。 在此之前，我只学过 Java 基础（数据类型、循环、面向对象）， 完全没有接触过 Spring Boot、MyBatis、MySQL 和 Redis。

🚀 先跑通，再理解——这是这个暑假最重要的认知。

  • 基于若依框架完成后端环境搭建

三、我遇到的 7 个问题

1. Maven 依赖下载慢 / 失败

现象：IDEA 打开项目后，右下角进度条转了 20 分钟还没停，还报了一堆红字。
解决：换成阿里云镜像源。修改 settings.xml，添加 mirror 配置。
学到：Maven 的依赖管理机制，以及国内镜像的重要性。

2. MySQL 连接失败：Access denied

现象：若依启动时报 Access denied for user 'root'@'localhost'。
解决：配置文件 application-druid.yml 里的密码和本地 MySQL 密码不一致，改成一致就好了。
学到：配置文件优先级、@Value 注入原理。

3. 数据库表不存在：Table 'ry-vue.sys_config' doesn't exist

现象：若依启动报错，说 sys_config 表不存在。
解决：在 sql 目录下找到 ry_2024xxxx.sql 和 quartz.sql，在 MySQL 里执行。
学到：如何用 MySQL 命令行执行 SQL 脚本，以及数据库初始化流程。

4. 接口 401 认证失败

现象：写的 /api/ai/chat 接口返回 401。
解决：在 SecurityConfig.java 的 permitAll() 列表里加上 /api/**。
学到：Spring Security 的拦截机制，以及如何配置白名单路径。

5. 接口 500：Content-Type 不支持

现象：放行后报错 Content-Type 'application/octet-stream' is not supported。
解决：Postman 的 Headers 里没设置 Content-Type: application/json，加上就好了。
学到：HTTP 请求头的作用，以及 Postman 的正确配置。

6. productId 始终为 null

现象：数据库里 product_id 有值，但接口返回的 productId 一直是 null。
解决：在 Mapper 里加 @Results 显式指定字段映射关系。
学到：MyBatis 的 ORM 映射原理，以及驼峰命名转换规则。

7. Redis 连不上

现象：启动报 Unable to connect to Redis。
解决：找到 Redis 安装目录，双击 redis-server.exe 启动。
学到：若依使用 Redis 做缓存，启动后端前必须先启动 Redis。

五、给同样在路上的同学

  • 先跑通，再理解。 不要试图一开始就搞懂每一行代码。先把项目跑起来，看到效果，再回头研究原理。
  • 报错是老师。 每一个红色的报错，都在教你一个具体的技术细节。复制报错信息去搜，比看 10 篇教程都管用。
  • 用 Postman 先验证。 不要等前端页面写好了再调接口。Postman 就是你的“前端”，先把接口测通，后面联调会顺利很多。
  • 不会 Python 也能做后端。 任务书里写的是 Python，但我用 Java 全部实现了。技术只是工具，重要的是功能完成。
